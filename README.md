# ブロック タイピング (kidtype)

7さいからの タイピングれんしゅう。Minecraft 風のブロック世界で、キーボードの位置を覚えるところから、
ローマ字 → 単語 → 文章へと、**自信がついた段階で順々に**進む。

- HTML5 + Canvas 2D + Web Audio API + PWA、**ビルド不要**（素の ES モジュール）。
- 対象端末: **iPad + 外付けキーボード（US 配列）**。画面のキーボード図が実物のキー位置に対応。

## フロー

**タイトル(ステージ自由選択) → ラウンド(有限の課題) → 結果(スコア画面)**

- タイトルで4つのステージを自由に選ぶ(タップ or 1〜4キー)。ベスト★・スコアを各カードに表示。
- 1ラウンド = 集中力が続く程度の固定課題数(`ROUND_COUNT` = キー16/ローマじ14/たんご8/ぶん5、調整可)。
- こなし切ると**結果画面**: ⏱じかん・🎯せいかい(正解率)・⭐スコア・★(1〜3)。`時間×正確性`でスコア算出、
  ベストは localStorage 保存(`kidtype:best:<stage>`)、更新で「しんきろく！」。「もういちど / えらぶ」。

### ステージ

1. **キー** — 光ったキーを探して1つ叩く（キー位置の体得）
2. **ローマじ** — かな1文字 → ローマ字（教える形は し=shi, ち=chi, つ=tsu, ふ=fu, じ=ji, ん=nn）
3. **たんご** — ひらがな/カタカナの単語
4. **ぶんしょう** — みじかい文（語間スペースは打鍵不要・自動スキップ）

**ヒントのラベルは出さない**（ローマ字ヒント・説明文なし）。頼りは**光るキーだけ**＝オンスクリーン
キーボードが教材。`canonicalNext()` が次の1キーを光らせ、`shi` のような複数打鍵もキー単位で誘導する。

## 動かし方（WSL → iPad over LAN）

```bash
# キャッシュ無効化ヘッダ付きの開発サーバ（変更が iPad に即反映される）。8011 で待受。
python3 serve.py 8011        # = npm run serve
```

iPad の Safari で **`http://192.168.1.5:8011/`** を開く。
（8000 は別アプリが使用中だったため 8011。`serve.py` は no-store ヘッダを付けるのでモジュールがキャッシュされない。）

- 「ホーム画面に追加」でフルスクリーン（standalone）になり、子ども向けに最適。
- **オフライン化（Service Worker）は HTTPS か localhost でのみ有効。** LAN の http では SW 登録を graceful にスキップ
  する（`main.js`）。本番は静的ホスティング（GitHub Pages / Netlify 等、HTTPS）にデプロイすると PWA オフラインが効く。

## テスト

```bash
node test/run.mjs   # エンジンの単体テスト（依存なし・自前ランナー）
```

ローマ字テーブル、入力マッチャ（NFA）の全特殊ケース（shi/si・ん の n/nn 文脈・促音・長音・拗音・カタカナ・
文中スペース自動スキップ・誤キー拒否）、習熟/段階解禁モデルを網羅。

## 構成

```
index.html / css/style.css / manifest.webmanifest / sw.js   … PWA シェル
src/
  main.js                  … 画面(タイトル/プレイ/結果)・入力・レイアウト・描画・スコア統括
  engine/
    kana.js                … カタカナ→ひらがな正規化など
    romaji.js              … かな→許容ローマ字・チャンク分解（表示メタ text/ci0/ci1 付き）
    matcher.js             … 入力マッチャ（チャンク列を NFA として走らせる）
    progress.js            … 習熟/適応出題モデル（※現フローでは未使用。将来の適応機能用に保持・テスト済）
    content.js             … 段階ごとの教材（順序付き）
  render/
    scene.js               … Minecraft 風の世界・採掘演出・パーティクル
    keyboard.js            … ブロック風 US キーボード（次キーを強調・指ゾーン色分け）
  audio/sfx.js             … Web Audio 合成 SFX（遅延 ctx・リミッタ・suspend ミュート）
test/run.mjs               … エンジンテスト
assets/icons/              … PWA アイコン（icon.svg から生成）
```

## 設計メモ

- **engine-driven**: 面白さの中枢は「正しく叩く→気持ちいいフィードバック→次の段階」のループ。
  ローマ字エンジンと入力マッチャを最初にテストで固めてから UI を載せた。
- ローマ字は **1つの形を教え、複数の形を受理**（shi も si も通すが、表示・指導は shi）。
- 入力マッチャは NFA。`n`/`nn`、`shi`/`si` の分岐や文中スペースを、ハード確定せず吸収する。
